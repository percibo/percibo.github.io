как сделать свой сайт с нуля

    Что понадобится:
  • время
  • любая электронная почта
  • папка, куда браузер складывает все загруженные файлы

1. Сайт начинается с файла index.html. Без всего остального можно обойтись, но этот файл и является началом сайта. Его можно скачать здесь . По умолчанию браузер помещает все загруженные файлы в одну папку, это можно узнать и поменять в настройках браузера. У меня это папка downloads. Находим там файл index.html, сначала открываем в браузере, чтобы посмотреть, как он выглядит сам по себе.

Потом открываем в любом текстовом редакторе, и в нем же можно отредактировать. Что значит каждый тег - можно посмотреть в интернете. Здесь как раз потребуется ваше время и желание: насколько глубоко вы хотите разобраться в вопросе. Всё можно поменять и настроить вручную.

2. Загружаем второй файл - картинку apple.jpg.

3. Нажимаем кнопку "обновить" в окне браузера - теперь картинку тоже стало видно.

4. Загружаем файл со стилями style.css

5. Нажимаем кнопку "обновить" в окне браузера - теперь появились рамка и поля.

6. Первоначальный сайт готов, его теперь можно выкладывать на обозрение всей планете. Для этого нам нужен хостинг - некий удаленный компьютер, постоянно включенный и подключенный к интернету. В этом компьютере будут находиться наши файлы. Один из вариантов бесплатного хостинга - это сайт github.com - при нажатии откроется в новом окне. Для регистрации нужна будет существующая электронная почта, придумать пароль и имя пользователя. Сайт на английском. Для образца я создала логин myawesomesite. Именно этот логин уже не доступен для использования - он теперь занят. Регистрируемся.

7. Create a repository - создаем репозиторий.

8. В поле Repository name указываем ваш логин пользователя гитхаб, затем ".github.io" (в образце - myawesomesite.github.io). Именно эта строчка - myawesomesite.github.io - будет адресом сайта, который можно ввести в окно браузера на любом устройстве по всему миру. И нажимаем Create repository.

9. В следующем окне выбрать uploading an existing file - "загрузить существующий файл", они у нас уже есть. Можно перетащить все 3 файла в открывшееся окно:

  • index.html
  • apple.jpg
  • style.css

И нажать кнопку внизу - Commit changes.

10. Готово! Вводим в браузере myawesomesite.github.io или даже https://myawesomesite.github.io/ и видим яблочные цветы.

Когда-то давно знали буквы, умели писать и читать единицы. Теперь скорее удивительная редкость те, кто это не делает.

Когда появились легковые четырехколесные машины, их водили только пилоты и те, у кого это стало занятием на всю их жизнь. Сегодня такую машину водят все, разве что кроме меня.

Так же и с сайтами. Свой базовый сайт способен сделать каждый человек, который сумел завести себе электронную почту, может написать что-то в текстовом файле и сделать фотку.

Администрировать шаблон это не есть писать сайт. Это больше похоже на езду в машине с водителем. У меня в этот момент навыки вождения не прокачиваются.

Чем больше посредников, тем меньше ты понимаешь сам и меньше можешь сделать и изменить. Пока что в моем сайте всего 2 посредника. Первый - это хранитель реестра доменных имен - я купила домен percibo.top. Второй - это хост.
